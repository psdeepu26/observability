📊 3 Pillars of Observability

Observability — Like you're 5, then like you're an engineer
🧒 LIKE YOU'RE 5
Imagine you're a doctor in an ER and a patient comes in feeling sick. You need 3 things:

❤️ Vitals (Metrics) — Heart rate, temperature, blood pressure. Numbers you track over time. "Heart rate spiked to 140 at 3pm" — tells you something happened.

📋 Patient History (Logs) — "What did you eat? Any allergies? When did the pain start?" Detailed timestamped records.

🔬 CT Scan (Traces) — A tiny camera through the body to see exactly which organ is failing. Follows one request through every system it touches.

The big idea: Metrics tell you something is wrong. Logs tell you what happened. Traces tell you exactly where. You need ALL THREE.
↓ Now let's call it what it really is ↓
🔧 FOR THE ENGINEER
❤️

Metrics

Numeric data over time.

Types: Counter (total requests), Gauge (current CPU), Histogram (latency distribution).

Tools: Prometheus, Grafana, Thanos, CloudWatch.

PromQL: rate(http_requests_total[5m])

📋

Logs

Timestamped immutable event records.

Structured > unstructured: JSON logs win. {"level":"error","latency":2.3}

Tools: Loki (k8s-native), ELK, Splunk, Datadog.

LogQL: {namespace="prod"} |= "error"

🔬

Traces

Distributed tracing across services.

Span: A unit of work ("DB query: 40ms").
Trace ID: Propagated via headers (traceparent).

Tools: Jaeger, Tempo, Zipkin, X-Ray.

OTel: tracer.startSpan("query_db")

🔗

The Holy Trinity

Grafana LGTM Stack:
• Loki (logs)
• Grafana (visualize)
• Tempo (traces)
• Mimir (metrics)

OpenTelemetry: Unified standard for all 3 signals.

🎯

USE / RED Methods

USE (Infra): Utilization, Saturation, Errors — per resource. CPU: util=75%, sat=runqueue, err=throttling

RED (Services): Rate, Errors, Duration — per request. API: rate=1000rps, err=0.1%, dur=p99=200ms

⚠️

Common Pitfall

"We collect metrics but never look at logs or traces."

Metrics are aggregated — they tell you a pod is OOM but not WHICH request. You need traces to find the culprit and logs for the details. No single pillar is enough.

⚡

Micro-action: Go to your Grafana and find a dashboard with only metrics. Can you trace a 500 error to a specific request and log entry? If not, you're missing pillars.
